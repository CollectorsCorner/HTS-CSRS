Type definition of biocompute objects Descriptions: 'line': refers to line types in the biocompute object. Line types preceeded by '_' means that the field is not mutable by a user who is running the biocompute object. 'name': refers to human readable name of the line type. This field can be empty. 'note': refers to description of the line type. ________________________________________________________________________________
line: _id name: id note: This is the biocompute object id. The id is a unique identifying value assigned to each object. This value is used for all references to this object.
line: _type name: Type note: This field defines the type of the object.
line: name name: Name note: The user-defined name of the object. It can be any type of text.
line: description name: Description note: The creator-specified description of the object or field in a type. Information in this field lets other users know the function and purpose of the object.
line: validation name: Validation note: Describes the position of an object in the validation process. The 'unreviewed' flag indicates that the object has been submitted, but no further evaluatation or validation has occurred. The 'in_progress' flag indicates that validation is underway. the 'reviewed' flag indicates that the biocompute object has been validated and reviewed. The 'manual' flag indicates that the object has been manually validated. The 'suspended' flag indicates an object that was once valid is no longer considered valid. The 'error' flag indicates that an error was detected with the biocompute object.
line: creation_date name: Creation Date note: The time and date that the biocompute object is created.
line: modification_date name: Modification Date note: This field will list all the dates and time when an object is modified.
line: publication_status name: Publication Status note: This is a choice field with four options. The 'draft' flag indicates that an object is in draft form and is still being edited. The 'open access' flag indicates that an object has been published and is freely available to anyone. The objects with the 'private' flag have restrictions on who can view and access them. This is a way for researchers using restricted data or metadata to insure the confidentially is maintained.
line: keywords name: Keywords note: Keywords to aid in search-ability and description of the object. These is a set of controlled vocabulary.
line: usability_domain name: Usability Domain note: Usability domain along with keywords can help determine when and how the biocompute object can be used. Novel use of the biocompute object should result in the creation of a new entry.
line: authors name: Authors note: Authors who created the initial biocompute object.
line: pipeline name: Pipeline note: Pipeline that is used to run the biocompute object.
line: pipeline, name name: Pipeline Name note: The name of the pipeline that is used to run the biocompute object.
line: pipeline, version name: Pipeline Version note: Version of the pipeline
line: pipeline, id name: Pipeline Id note: Pipeline ID.
line: svc-align-hexagon name: HIVE-Hexagon DNA Sequence Alignment note: Arguments for alignment using Hexagon. HIVE-hexagon is an alignment algorithm that finds short read alignments by seeding, extension and optimal alignment. The hexagon algorithm runs faster than other industry favorite-algorithms and therefore is able to use a smaller seed size, increasing sensitivity without sacrificing performance.
line: row name: Row note: Row indicates the role a field has in the computation. If the field is a parametric field then it will not have a row value. If it is an input or output value then it will have those flages, respectivly.
line: svc-align-hexagon, minimum_match_len name: Minimum Match Length note: Minimum match length needed to be considered for the alignment.
line: svc-align-hexagon, mismatch_allowed name: Mismatch Allowed note: Mismatches allowed to be considered for the alignment.
line: svc-dna-heptagon name: HIVE-heptagon Profiling Tool note: Arguments for variant calling. Heptagon allows the user to view and download summaries, profile graphs, SNP calls. Profiling results are summarized in the interface in various tables and charts and also through a variety of downloads.
line: svc-dna-heptagon, min_coverage name: Varient Calling Minimum Coverage note: Minimum depth of read coverage required for variant calling.
line: svc-dna-heptagon, freq_cutoff name: Varient Calling Frequency Cutoff note: Percent of reads that should have the single nucleotide variation for variant calling.
line: subject name: Reference Sequences note: Nucleotide sequences to which the NGS reads are to be mapped.
line: query name: Read File note: Nucleotide reads to be alinged.
line: snv_profile name: SNV Profile note: Output of variant calling.
line: alignment_file name: Alignemnt File note: Output of alignment.
line: aminoacid_calls name: Amino Acid Calls note: Output of amino acid calls.
